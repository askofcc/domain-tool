✦ AI Agent 智能体 & API 接口调用指南

本系统支持 AI 智能体（Agent）或自定义脚本直接发起批量域名查询。由于后端 Worker 已自动处理了 Session 维持和 CORS 跨域，调用方无需关注 Cookie 或 Referer 伪装，直接向本服务接口发送标准请求即可：

1. 接口规范

  • 接口地址：/api/check
  • 请求方式：POST
  • 请求数据类型：application/json
  • 请求参数（JSON 结构，最大支持 60 个域名）： 
    {
  "domains": ["apple.ai", "banana.io", "google.com"]
}
  • 返回 JSON 结构（接口返回本轮可确认状态；低压策略下暂未确认会返回 timeout，可稍后重试）： 
    [
  {"status":"success","result":"available","domain":"apple.ai"},
  {"status":"success","result":"unavailable","domain":"banana.io"},
  {"status":"success","result":"unavailable","domain":"google.com"}
]

2. AI Agent Tool (工具调用) 提示词与架构设计

在大模型中，您可以直接将本接口声明为如下格式 of Tool：

{
  "name": "check_domains",
  "description": "批量查询域名的可用性（是否未注册）。支持输入以逗号分隔的多个域名。",
  "parameters": {
    "type": "object",
    "properties": {
      "domains": {
        "type": "array",
        "items": { "type": "string" },
        "description": "域名列表数组，例如: [\"getabc.ai\", \"gochat.io\", \"mycode.com\"]"
      }
    },
    "required": ["domains"]
  }
}

AI 执行策略建议：

  1. 规则翻译：当用户说“我想找个 3 位数字的 .ai 域名”时，大模型应当自动生成全部目标域名列表（如 000.ai, 001.ai, ..., 999.ai）。
  2. 低压调用：大模型或脚本按最多 60 个域名一批调用即可；遇到 timeout 代表本轮暂未确认，建议稍后重试，避免连续高频轮询触发上游限流。
  3. 价值精选：大模型拿到可用（available）域名列表后，可基于自身知识库进行商业价值精选（如优先挑出包含完整单词、纯数字或押韵的域名），提供高质量的推荐理由。

3. 终端命令行快速查询 (cURL 示例)

您也可以在本地终端或编写自定义脚本直接通过 curl 批量查询域名（已自动处理跨域和请求头伪装）：

curl -X POST http://localhost:5173/api/check \
  -H "Content-Type: application/json" \
  -d '{"domains":["apple.ai","google.com","testxyz12345.com"]}'

接口会在云端先执行 DNS 预过滤，再进行低频 WHOIS 查询，以 JSON 格式返回本轮可确认的域名状态。
